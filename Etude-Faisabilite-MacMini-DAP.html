1 Installation et configuration du Mac Mini comme serveur

1.1 Choix du matériel

Apple propose actuellement plusieurs configurations de Mac Mini M4. Pour notre usage (services multiples + LLM locaux), voici la comparaison :

Configuration Puce RAM Stockage Prix Adapté pour
M4 Base M4 (10 cœurs) 16 Go 256 Go 649 € Usage bureautique, pas de LLM
M4 — 16/512 M4 (10 cœurs) 16 Go 512 Go 879 € Services légers, LLM 7B limités
M4 Pro — 24/512 M4 Pro (12 cœurs) 24 Go 512 Go 1 649 € Multi-services + LLM 13B
M4 Pro — 48 Go / 1 To ★ M4 Pro (14 cœurs) 48 Go 1 To 2 199 € LLM 30-70B, multi-services, evolutif
Recommandation pour DAP : M4 Pro 48 Go / 1 To à 2 199 € Cette configuration offre la puissance nécessaire pour faire tourner simultanément nos outils (n8n, bases de données, applications Docker) et des LLM performants (jusqu'à 30-70B paramètres). Avec 48 Go de RAM unifiée et la bande passante mémoire du M4 Pro (273 Go/s), c'est un investissement pérenne qui nous évite de devoir upgrader dans 1-2 ans. Le 1 To de stockage laisse de la marge pour les modèles IA (plusieurs Go chacun) et les données.

1.2 Étapes d'installation — Guide pas à pas

Étape A : Configuration initiale macOS

1
Premier démarrage — Brancher un écran/clavier, configurer macOS normalement. Créer un compte administrateur (ex : admin-dap).
2
Désactiver la mise en veille — Le serveur doit rester allumé 24/7. 
# Désactiver toute mise en veille
sudo pmset -a sleep 0
sudo pmset -a disksleep 0
sudo pmset -a displaysleep 0
3
Redémarrage auto après coupure de courant 
sudo pmset -a autorestart 1
Ou via Réglages Système → Général → Démarrage et arrêt.
4
Activer l'ouverture de session automatique — Réglages Système → Utilisateurs et groupes → Ouverture de session automatique. Le Mac redémarrera sans intervention humaine.

Étape B : Accès distant (SSH + VNC)

5
Activer SSH — Pour l'accès en ligne de commande à distance. 
# Activer le service SSH
sudo systemsetup -setremotelogin on
Ou via Réglages Système → Général → Partage → Session à distance.
6
Activer le Partage d'écran (VNC) — Pour un accès graphique à distance. 
# Activer VNC
sudo /System/Library/CoreServices/RemoteManagement/ARDAgent.app/Contents/Resources/kickstart \
  -activate -configure -allowAccessFor -allUsers \
  -configure -clientopts -setvnclegacy -vnclegacy yes \
  -configure -clientopts -setvncpw -vncpw "MOT_DE_PASSE_VNC" \
  -restart -agent -privs -all
Connexion depuis un autre Mac : vnc://IP_DU_MAC_MINI:5900
7
Adaptateur HDMI factice (~10 € sur Amazon) — Sans écran branché, macOS limite la résolution. Ce petit dongle simule un écran 4K pour que le partage d'écran fonctionne correctement.

Étape C : Installation des logiciels de base

8
Installer Homebrew — Le gestionnaire de paquets pour macOS. 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
9
Installer OrbStack (remplacement de Docker Desktop, plus léger et performant). 
brew install orbstack
# Lancer OrbStack
open -a OrbStack
# Vérifier que Docker fonctionne
docker version
10
Installer Cloudflare Tunnel — Pour exposer les services sur Internet en toute sécurité, sans ouvrir de ports sur le routeur. 
brew install cloudflare/cloudflare/cloudflared
# Se connecter à Cloudflare
cloudflared tunnel login
# Créer le tunnel
cloudflared tunnel create mac-mini-dap
# Configurer le DNS (répéter pour chaque sous-domaine)
cloudflared tunnel route dns mac-mini-dap service1.votre-domaine.com
# Installer comme service au démarrage
sudo cloudflared service install

2 Implémentation des outils et LLM

2.1 Déploiement des outils internes via Docker

Grâce à Docker (OrbStack), chaque outil tourne dans son propre conteneur isolé. L'ajout, la mise à jour ou la suppression d'un service se fait sans impacter les autres. Voici les types de services que le Mac Mini peut héberger :

Type de service Exemples Faisable sur M4 Pro 48 Go
Automatisation / workflows n8n, Node-RED, Activepieces Excellent
Base de données PostgreSQL, MySQL, Redis Excellent
IA / LLM local Ollama, LM Studio Excellent (jusqu'à 70B)
API internes APIs Node.js, Python, Go Excellent
Monitoring Uptime Kuma, Glances Excellent
Serveur web Nginx, Caddy Excellent
Serveur mail — Non recommandé (délivrabilité)

Le principe est simple : chaque outil a son fichier docker-compose.yml, se lance avec docker compose up -d, et est exposé sur Internet via Cloudflare Tunnel (un sous-domaine par service). L'ajout d'un nouveau service prend quelques minutes.

2.2 LLM locaux avec Ollama

Ollama permet d'exécuter des modèles d'IA (LLM) directement sur le Mac Mini, sans passer par des API cloud payantes. Avec 48 Go de RAM, la gamme de modèles utilisables est très large :

Performances sur Mac Mini M4 Pro — 48 Go

Modèle Taille Vitesse Verdict
Phi-3 Mini 3.8B 60-80 tokens/s Excellent
Llama 3.1 / Mistral 7B 7-8B 40-55 tokens/s Excellent
Qwen2 14B 14B 25-35 tokens/s Très bien
DeepSeek R1 32B 11-14 tokens/s Bon (raisonnement avancé)
Llama 3.1 70B 70B 5-8 tokens/s Utilisable (lent)
Mixtral 8x7B 47B 8-12 tokens/s Correct
Que signifient ces chiffres ? 25-40 tokens/seconde ≈ lecture d'un texte à vitesse normale. Un modèle 7B comme Mistral est capable de répondre à des questions, résumer des documents, analyser des appels. Un modèle 32B comme DeepSeek R1 offre des capacités de raisonnement proches des meilleurs modèles cloud. Le facteur clé est la bande passante mémoire du M4 Pro (273 Go/s), bien supérieure au M4 standard (120 Go/s).

Installation d'Ollama

# Installer
brew install ollama

# Démarrer le service
ollama serve

# Télécharger un modèle (première fois uniquement)
ollama pull mistral:7b        # ~4 Go
ollama pull deepseek-r1:32b   # ~19 Go

# Tester
ollama run mistral:7b "Résume en une phrase : DAP est un courtier en assurances."

# L'API REST est disponible sur http://localhost:11434
# → Utilisable depuis n'importe quel outil via un appel HTTP

2.3 Approche hybride recommandée

Plutôt que de tout centraliser en local, nous recommandons une approche hybride — le Mac Mini pour ce qui a besoin de puissance, contrôle ou confidentialité ; le cloud pour le reste :

Service Hébergement Raison
Outils d'automatisation (n8n, etc.) Mac Mini (local) Contrôle total, pas de limite d'exécution
Bases de données Mac Mini (local) Données en local = RGPD simplifié
LLM locaux (Ollama) Mac Mini (local) Pas de coût par requête, données restent locales
APIs internes Mac Mini (local) Performance, latence faible
Claude API (tâches complexes) Cloud (Anthropic) Quand un LLM local ne suffit pas
Téléphonie (Twilio, Telavox) Cloud Services cloud par nature
Email (Outlook) Cloud Ne jamais héberger un serveur mail

2.4 Test minimal avant migration complète

Oui, on peut tester une version minimale d'abord
  1. Installer un ou deux services Docker sur le Mac Mini (ex : un outil d'automatisation + une base de données)
  2. Vérifier que Cloudflare Tunnel fonctionne (accès depuis l'extérieur)
  3. Tester Ollama avec un prompt simple
  4. Valider la stabilité pendant 1 semaine
  5. Migrer progressivement les autres services

3.2 Sauvegardes automatisées

Un script de backup tourne chaque nuit à 3h du matin, sauvegardant les bases de données et les volumes Docker.

Stratégie de backup en 3 niveaux :

Niveau Outil Fréquence Destination
1. Backup applicatif Script bash (cron) Quotidien 3h Disque local
2. Backup système Time Machine Toutes les heures Disque USB externe
3. Backup hors-site Restic → Backblaze B2 Quotidien (nuit) Cloud (~5 €/mois)
Point critique pour les services avec chiffrement Certains outils (comme n8n) utilisent une clé de chiffrement pour protéger les credentials stockés. Si cette clé est perdue, les credentials deviennent illisibles. Toujours sauvegarder les clés de chiffrement séparément (gestionnaire de mots de passe, coffre-fort).

5.2 Plan B — Solution cloud hybride

Plan B en cas de problème majeur avec le Mac Mini

Si le Mac Mini tombe en panne durablement, nous pouvons redéployer les services sur un VPS en 1-2 heures :

  1. Louer un VPS (Hostinger, Hetzner, OVH) — opérationnel en 5 min
  2. Installer Docker + restaurer les backups depuis Backblaze B2
  3. Mettre à jour le DNS Cloudflare pour pointer vers le nouveau serveur
  4. Les services reprennent — seules les données entre le dernier backup et la panne sont perdues (max 24h)

C'est l'un des grands avantages de Docker : l'infrastructure est portable. Les mêmes fichiers docker-compose.yml fonctionnent sur n'importe quel serveur.

6 Recommandations et prochaines étapes

6.1 Recommandation

Go — Nous recommandons de lancer le projet

Le Mac Mini M4 Pro 48 Go est une solution fiable, puissante et adaptée aux ambitions de DAP. L'investissement (~2 400 €) est maîtrisé, le risque est couvert grâce à Cloudflare Tunnel et aux backups multi-niveaux. Avec 48 Go de RAM et 1 To de stockage, cette machine peut héberger l'ensemble de nos outils actuels ET futurs, y compris des LLM performants en local.

6.3 Démonstration / Pilote

Oui, un pilote est prévu (Semaine 2)

Avant de migrer quoi que ce soit en production, nous ferons tourner le Mac Mini en parallèle du VPS Hostinger pendant au moins une semaine. Cela permet de :

  • Valider la stabilité de la connexion Internet du bureau
  • Vérifier le bon fonctionnement de Cloudflare Tunnel
  • Tester des services Docker de bout en bout
  • S'assurer que le redémarrage automatique fonctionne après coupure de courant
  • Mesurer les performances d'Ollama sur un cas réel

Aucun risque pour la production : le VPS reste actif tant que le pilote n'est pas concluant.

Annexe — Configuration sécurité détaillée

Firewall PF (protection anti-brute-force)

# Ajouter dans /etc/pf.conf

# Table pour les IPs bannies automatiquement
table <bruteforce> persist

# Bloquer les IPs bannies
block quick from <bruteforce>

# SSH : max 3 tentatives en 30 secondes, sinon ban
pass in on en0 proto tcp to port 2222 flags S/SA \
  keep state (max-src-conn 5, max-src-conn-rate 3/30, \
  overload <bruteforce> flush global)

# Activer
sudo pfctl -f /etc/pf.conf
sudo pfctl -e

SSH — Authentification par clé uniquement

# Sur votre ordinateur de travail : générer une clé
ssh-keygen -t ed25519 -C "admin@dap"

# Copier la clé sur le Mac Mini
ssh-copy-id -p 2222 admin-dap@IP_MAC_MINI

# Sur le Mac Mini : désactiver l'authentification par mot de passe
# Éditer /etc/ssh/sshd_config :
Port 2222
PasswordAuthentication no
PermitRootLogin no
AllowUsers admin-dap

Mises à jour — Configuration recommandée

# Activer les mises à jour de sécurité automatiques
sudo defaults write /Library/Preferences/com.apple.SoftwareUpdate CriticalUpdateInstall -bool true

# Désactiver les mises à jour majeures automatiques
sudo defaults write /Library/Preferences/com.apple.SoftwareUpdate AutomaticallyInstallMacOSUpdates -bool false

# Garder le téléchargement automatique (pour être prêt)
sudo defaults write /Library/Preferences/com.apple.SoftwareUpdate AutomaticDownload -bool true
